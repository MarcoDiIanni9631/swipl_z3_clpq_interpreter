1. Interprete per smart contracts

   * Simula il contratto in Prolog.
   * Salta funzioni non rilevanti.
   * Verifica se un `assert` può fallire (con Z3).

2. Generazione casi di test

   * Ogni percorso → un caso di test.
   * SAT → test che causa errore.
   * UNSAT → percorso sicuro.
   * Output: set di test concreti.

3. Report finale

   * Mostra percorsi, profondità, loop, esiti SAT/UNSAT.
   * Riassume vincoli e condizioni raggiungibili.
   * Include statistiche (copertura, tempo, complessità).
